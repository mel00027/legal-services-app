# Огляд проєкту: LegalClick (Fintech Legal Platform)

Цей файл створений для швидкого занурення в проєкт для штучного інтелекту або нових розробників без необхідності читати весь вихідний код.

## 1. Загальна архітектура
Проєкт складається з двох основних частин:
1. **Frontend (Landing Page Web App)**: Побудований на React (Vite) з використанням TailwindCSS.
2. **Backend (Telegram Bot)**: Побудований на Node.js з використанням бібліотеки `telegraf`.

---

## 2. Віджет на Frontend (d:\stitch\UI\legal-services-app)
- **Технології:** React 19, React Router, TailwindCSS v4, Framer Motion (для анімацій), Lucide React (для іконок).
- **Стилістика:** Dark Mode фінтех-додаток (глибокі чорні/сірі кольори, яскравий синій акцент `#0052FF`, ефекти Glassmorphism).
- **Головний файл:** `src/App.jsx`
- **Основні логічні блоки Landing Page:**
  - Навігаційний хедер з налаштованою системою плавного скролу (`scroll-behavior: smooth`, фікс `scroll-padding-top` у `index.css`).
  - Hero-блок із закликом до дії "Чат з юристом".
  - Конверсійні картки "Чому ми?" (унікальні переваги).
  - Вертикальний (мобайл) та горизонтальний (десктоп) timeline "Як це працює".
  - Блок "Напрямки послуг" з іконками категорій.
  - Тарифна картка "Вхідний квиток" з фіксованою ціною (500 грн).
  - Блок FAQ (Акордеон) з доданою SEO-мікророзміткою `Schema.org/FAQPage`.
- **Запуск Frontend:** `npm run dev` (швидкий гарячий сервер Vite).
- **SEO & Аналітика:** Додані базові сніпети (placeholders) для Google Analytics та Meta Pixel. Кнопки `Call to Action` використовують протокол `tg://resolve?domain=your_bot_name` для глибинних лінків на бота.

## 3. Бот на Backend (d:\stitch\UI\legal-services-app\server)
- **Головний файл:** `server/index.js`
- **Особливість роботи:** Бот підключається до групи адміністраторів (Forum), що підтримує "Топіки" (Topics).
- **Бізнес-логіка (Воронка користувача):**
  1. Стартове меню (`/start`): Вибір юридичної послуги через Inline-кнопки (Категорія).
  2. Запит опису проблеми: Одразу після вибору категорії користувач вводить текст, надсилає голосове повідомлення або документи (`awaiting_description`).
  3. Вибір методу оплати: [💳 Прямий переказ] або [💎 USDT Крипта].
  4. Обробка фіатної оплати:
     - Клієнт отримує реквізити та надсилає квитанцію (`awaiting_receipt`).
     - **Таймаут:** Працює автоматичний таймер, заявки в очікуванні квитанції понад 30 хвилин автоматично скасовуються.
     - Бот пересилає всю інформацію в "Загальний" чат адмінів (Форум) з кнопками `[✅ Підтвердити]` або `[❌ Відхилити]`.
  5. Підтвердження оплати: При кліку адміном на `✅ Підтвердити`, бот АВТОМАТИЧНО створює окремий Топік (Thread) для цього клієнта.
  6. Комунікація (Двосторонній зв'язок): 
     - Будь-яке повідомлення від клієнта пересилається у створений для нього Топік.
     - Відповіді юристів у цьому Топіку відправляються клієнту від імені бота.
- **Модуль керування діалогом (Адмін-панель у Топіку):**
  - При створенні Топіка бот відправляє перше повідомлення із даними заявки та ЗАКРІПЛЮЄ його (Pin).
  - Під цим повідомленням є кнопки керування: 
    - `[💰 Виставити рахунок на доплату]` (Бот питає суму та відправляє клієнту інвойс).
    - `[✅ Завершити консультацію]` (Закриває сесію).
- **Обробка помилок (Error Boundary):** Запроваджено `bot.catch` для глобального перехоплення помилок Telegram API.
  - **Авто-відновлення топіка:** Якщо адмін випадково видалив топік клієнта, бот перехоплює помилку `thread not found` під час пересилання повідомлення і автоматично створює новий топік для продовження переписки без падіння сервера.

## 4. Зберігання стану клієнтів (SQLite Database)
Після оновлення стейт-менеджера архітектуру переведено зі звичайної оперативної пам'яті (`Map`) на **SQLite** базу даних (`database.sqlite`).
- **`sessions`:** Зберігає поточний крок юзера (напр., `awaiting_receipt`) та тимчасові дані (категорія, сума доплати тощо). Це запобігає втраті даних клієнтів при перезапуску Node.js сервера.
- **`topics`:** Стійке збереження мапінгу `chatId` (клієнта) ↔ `threadId` (ID топіка).

---

## 5. Змінні оточення (.env у папці server)
- `BOT_TOKEN`: Токен основного бота.
- `ADMIN_CHAT_ID`: ID групи адмінів (з префіксом `-100`, має бути активована опція Forum).
- `CARD_NUMBER`: Номер карти для прямих переказів.
- `CRYPTO_PAY_TOKEN`: Токен для API CryptoBot (покращення на майбутнє).

## 6. Важливі особливості розробки
- Всі дії з навігацією працюють без `react-router-dom` шляхом нативних ID-якорів, оскільки проєкт - Single Page Application з лендинг-форматом.
- Щоб бот міг коректно створювати/пересилати в топіки, бот МАЄ мати адмін-права в групі!
